<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">在设定时间内向大语言模型发送请求的频率。速率限制能削弱攻击者</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">快速试验或发起集中攻击的能力，从而降低威胁。实施速率限制有多</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">种方式，每种方法都有其独特的优势：</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">基于IP的速率限制</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">此方法限制了来自特定IP地址的请求数量。它对于阻止来自单一位置</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">的个体攻击者特别有效，但可能无法全面防御利用多个IP地址的分布</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">式攻击。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">基于用户的速率限制</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">此技术将速率限制与已验证的用户凭据相关联，提供了一种更具针对</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">性的方法。它能够防止已验证用户滥用系统，但需依赖要已建立的认</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">证机制。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">基于会话的速率限制</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">此选项限制每个用户会话中允许的请求数量，非常适合用户与大模型</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">保持持续会话的Web应用程序。</span>

<span style="font-family:'Unnamed-T3';font-size:16.69pt;color:#000000">4.4.2</span><span style="font-family:'Unnamed-T3';font-size:16.69pt;color:#000000"> </span><span style="font-family:'Unnamed-T3';font-size:16.69pt;color:#000000">基于规则的输入过滤</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">基本的输入过滤是一个逻辑控制点，在阻止SQL注入等攻击方面有着</span>

<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">施的直接且合适的切入点，是防御提示词注入攻击的第一道合理防</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">线。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">与其他需要复杂系统架构更改的安全实现不同，输入过滤可以通过现</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">有的工具和规则集进行管理，因此实施起来相对简单一些。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">然而，提示词注入的独特性和复杂性使得使用传统的输入过滤方法来</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">解决这个问题特别具有挑战性。与SQL注入不同，后者可以通过精心</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">构建的正则表达式（regex）捕获大多数恶意输入，而提示词注入攻击</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">能够演变和适应，从而绕过简单的正则表达式过滤器。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">此外，这些简单的输入过滤规则可能会降低应用程序的性能。想象一</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">下我们在前面章节中讨论过的“祖母制作凝固汽油弹”的例子。最可</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">靠的防护措施可能是将“凝固汽油弹”和“炸弹”等词加入黑名单。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">然而，这也会严重削弱模型的功能，因为这会消除细微差别，导致其</span>

<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">大模型以自然语言解释输入，这本身就比结构化查询语言更复杂且多</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">变。这种复杂性使得设计一套既有效又全面的过滤规则变得十分困</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">难。因此，至关重要的是将输入过滤视为多层次安全策略中的一环，</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">并根据新出现的威胁调整过滤规则。</span>

<span style="font-family:'Unnamed-T3';font-size:16.69pt;color:#000000">4.4.6</span><span style="font-family:'Unnamed-T3';font-size:16.69pt;color:#000000"> </span><span style="font-family:'Unnamed-T3';font-size:16.69pt;color:#000000">悲观信任边界定义</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">鉴于提示词注入攻击的复杂性和不断演变的特性，一个有效的缓解策</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">略是在大语言模型周围实施一种悲观信任边界。这种方法承认了防御</span>

<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">示时，我们应将大模型的所有输出都视为本质上不可信的。</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">这种策略以一种更为怀疑的视角重新定义了信任的概念。它摒弃了那</span>
<span style="font-family:'Unnamed-T3';font-size:14.83pt;color:#000000">种认为配置良好的大模型能够自动过滤掉危险或恶意输入的假设，而</span>
