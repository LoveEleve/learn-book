# 第六个月 · 费曼检验

> **目标**：每月最后一个周末，随机抽取3个核心知识点，向一个完全不懂技术的人讲清楚。讲不出来 = 没学会。

## 检验规则

2. 对每个知识点进行15分钟讲解，不能用任何笔记

4. 如果卡住了——回到材料重新学习，24小时后再来

## 本月3个核心知识点（随机抽取）

### 1. Go GMP 调度模型
**讲述要求**：不用 Goroutine/M/P 这些 Go 术语，而是用"工厂流水线"比喻——订单（G，goroutine）等着被处理，工人（M，machine 线程）负责干活，工作站（P，processor）是工人干活的地方。关键设计：工作站比工人多→工人可以换工作站（work stealing）；工作站不够→新订单排队。对比 Java 线程模型（一个订单一个工人，开销大）。

**核心概念链**：G（goroutine，轻量级用户态协程，初始栈 2KB）→ M（machine，OS 内核线程，真正执行代码）→ P（processor，逻辑处理器，持有 G 的本地队列）→ GOMAXPROCS（P 的数量，默认 CPU 核数）→ 调度循环（schedule→execute→goexit→schedule）→ work stealing（M 绑定的 P 空了，从其他 P 偷一半 G）→ hand off（M 因系统调用阻塞，P 带着剩余 G 去找其他 M）。

**参考资料**：第二十一周/day1-2-Go并发模型.md、第二十二周/day1-2-Go接口反射.md

### 2. Transformer Self-Attention 机制
**讲述要求**：不是写公式，而是讲清楚"Self-Attention 解决了 RNN 的什么问题"。RNN 像一个人按顺序读一篇文章，读到最后一句时前面已经模糊了。Self-Attention 像一群人同时读，每个人（每个词）都和其他所有人"交换纸条"（计算注意力分数），这样谁和谁有关系就一目了然。

**核心概念链**：RNN 的问题（长距离依赖衰减、无法并行）→ Self-Attention 本质（每个词和所有词算相似度→加权求和得到上下文表示）→ Q（Query，我想找谁）、K（Key，我是谁）、V（Value，我的信息）→ 计算公式 `Attention(Q,K,V) = softmax(QK^T/√dk)V` → 除以 √dk（防止点积过大导致 softmax 梯度消失）→ Multi-Head（多头从不同角度关注不同关系）。

**参考资料**：第二十三周/day1-2-AI基础.md

### 3. Go GC 三色标记+混合写屏障 vs JVM G1 SATB
**讲述要求**：先用日常生活比喻讲清三色标记——白色（没见过的垃圾）、灰色（正在检查的）、黑色（确认有用的）——这是两种 GC 的共同框架。然后讲关键区别：G1 用 SATB（开始并发标记时拍快照），Go 用混合写屏障（插入屏障+删除屏障结合）。为什么它们各自选了不同方案？G1 有 Remembered Set 管理跨 Region 引用，Go 堆更简单不需要 RSet。

**核心概念链**：三色标记通用框架（白/灰/黑）→ 漏标条件（同时满足：黑色对象指向白色对象断开+灰色对象到白色对象新建引用）→ G1 方案：SATB（标记开始时逻辑快照，pre-write barrier 记录旧值）→ Go 方案：混合写屏障（Dijkstra 插入屏障+Yuasa 删除屏障，栈上对象特殊处理）→ Go 1.8 之前只用 Dijkstra 插入屏障（需要 STW 重新扫描栈）→ 1.8 加入混合写屏障（栈上对象变黑色时触发屏障，消除栈重扫 STW）。

**参考资料**：第二十一周/day3-4-Go内存与GC.md、第二个月/第五周/day3-4-SATB标记.md
