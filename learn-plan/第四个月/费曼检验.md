# 第四个月 · 费曼检验

> **目标**：每月最后一个周末，随机抽取3个核心知识点，向一个完全不懂技术的人讲清楚。讲不出来 = 没学会。

## 检验规则
1. 找一个真人听众（推荐）或用录音设备
2. 对每个知识点进行15分钟讲解，不能用任何笔记

## 本月3个核心知识点（随机抽取）

### 1. Netty 主从 Reactor 多线程模型
**讲述要求**：用"餐厅前台+服务员"比喻讲清楚——BossGroup（前台接待）只负责引导客人到座位（accept 连接），WorkGroup（服务员）负责接待每桌客人（处理 IO 读写）。一个连接绑定一个 Work 线程（无锁化），每个线程通过 epoll 同时照看多桌。对比单 Reactor 的瓶颈（接待+服务都是一个人）。

**核心概念链**：BossGroup（NioEventLoop 线程池）→ 每个线程有独立 Selector → 监听 ACCEPT 事件 → accept 后 Channel 注册到 WorkGroup 的某个 Selector → WorkGroup 线程负责 READ/WRITE 事件 → Pipeline 责任链（ChannelHandler 双向链表）→ 编解码→业务处理→编码→写回。

**参考资料**：第十三周/Netty核心.md、第一个月/第四周/day4-5-Reactor.md

### 2. Docker 三大核心技术原理
**讲述要求**：不是背 Namespace 有哪六种，而是讲清楚"为什么 Docker 可以在一台机器上跑几千个容器，互相看不到对方，又不能吃太多资源"——Namespace（隔离视线）、Cgroups（限制饭量）、UnionFS（共享基础食材，各做各的菜）。用"大学宿舍"比喻。

**核心概念链**：PID Namespace（进程隔离，容器内 PID 1 不是宿主机 PID 1）→ NET Namespace（网络隔离，每个容器有独立网卡/IP/端口空间）→ MNT Namespace（文件系统隔离）→ UTS/ IPC / USER Namespace → Cgroups（CPU shares/CFS 配额、memory.limit、blkio 限制）→ UnionFS（Overlay2: lowerdir 镜像层只读、upperdir 容器层可写、merged 统一视图、写时复制 COPY_ON_WRITE）。

**参考资料**：第十五周/Docker原理.md

### 3. Spring 三级缓存解决循环依赖
**讲述要求**：用"A 和 B 互相需要才能工作"的场景讲清楚——A 正在造，发现需要 B；去造 B，发现 B 需要 A；如果 A 还没完成就不能给 B，那就死循环了。Spring 的做法：先让 A "暴露自己还没完成的半成品"（三级缓存），B 拿着半成品继续造自己，完成后再把成品 A 替换掉。

**核心概念链**：一级缓存 singletonObjects（成品）→ 二级缓存 earlySingletonObjects（半成品，已实例化待填充）→ 三级缓存 singletonFactories（半成品工厂，可生成 AOP 代理）→ A 实例化→放入三级缓存→填充属性 B→创建 B→B 填充属性 A→从三级缓存获取 A 的早期引用→B 完成→放入一级→A 继续完成→放入一级。构造器注入无法解决（因为构造器就是实例化过程，没有"早期暴露"的机会）。

**参考资料**：第十三周/Spring核心.md、第十六周/day1-2-架构总结.md
