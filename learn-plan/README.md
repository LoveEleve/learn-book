## 五、关键学习建议

1. **核心书优先**：每阶段主攻 3-4 本核心书，其余为查阅/补充。读完一章就要能讲出来。
2. **先骨架后细节**：先通读建立脉络（第1遍），再精读核心章节并动手验证（第2遍）。
3. **多书交叉印证**：同一主题参考 2-3 本不同作者的书，从不同角度理解同一概念。
4. **输出驱动**：每学完一个主题，写一篇笔记/博客/分享。讲不出来 = 没学会。
5. **动手验证**：GDB 看 JVM 堆结构、tcpdump 抓包、JMH 基准测试——代码不会骗人。
6. **面试导向**：每个主题学完后问自己"面试官会怎么问这个问题"，准备 3 分钟回答。
