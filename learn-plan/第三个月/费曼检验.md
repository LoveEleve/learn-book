# 第三个月 · 费曼检验

> **目标**：每月最后一个周末，随机抽取3个核心知识点，向一个完全不懂技术的人讲清楚。讲不出来 = 没学会。

## 检验规则
1. 找一个真人听众（推荐）或用录音设备
2. 对每个知识点进行15分钟讲解，不能用任何笔记
3. 听众必须能复述50%以上才算通过
4. 如果卡住了——回到材料重新学习，24小时后再来

## 本月3个核心知识点（随机抽取）

### 1. MySQL 一条 SQL 的完整执行路径
**讲述要求**：从 `SELECT * FROM users WHERE id = 1` 开始，讲清楚这条 SQL 经历了什么。用"图书馆找书"比喻：解析器（理解你要找什么）→ 优化器（选最快的方式找）→ 执行器（去书架上取书）→ 存储引擎（书架管理系统真正翻页）。

**核心概念链**：连接器（认证+权限）→ 查询缓存（8.0 已移除）→ 分析器（词法+语法分析生成 AST）→ 优化器（索引选择、Join 顺序、代价估算）→ 执行器（调用存储引擎接口）→ InnoDB（B+树索引查找→Buffer Pool→读页→返回行）→ undo log（MVCC 回滚段）→ redo log（崩溃恢复）。

**参考资料**：第九周/day1-2-索引.md、第九周/day3-4-事务日志.md、第九周/day5-MVCC锁.md

### 2. CAP 理论与分布式事务取舍
**讲述要求**：不是背 CAP 的定义，而是用"异地恋"比喻：两个人（节点）断联了（网络分区 P）——要么坚持"我们必须每天视频"（一致性 C）但可能打不通电话（牺牲可用性 A），要么接受"有空再打"（可用性 A）但可能不知道对方今天干了什么（牺牲一致性 C）。然后讲清楚 2PC/TCC/Saga 分别适合什么场景（严格扣钱 vs 宽容退票）。

**核心概念链**：CAP 定理（P 必选，C 和 A 二选一）→ 实际是 PACELC（有分区时选 C 或 A，无分区时选 L 或 C）→ 2PC（强一致但阻塞）→ TCC（业务层补偿，Try/Confirm/Cancel）→ Saga（长事务链，前向恢复/后向补偿）→ Seata AT（自动回滚，undo_log 镜像）。

**参考资料**：第十一周/day1-2-CAP一致性.md、第十一周/day5-分布式事务.md

### 3. Redis 高性能数据结构与持久化
**讲述要求**：不要罗列 5 种数据结构，而是讲清楚"为什么 Redis 这么快"——不是"纯内存"一句话就完了。深入讲：为什么单线程还快？因为是 IO 多路复用 + 批量写入 AOF 策略 + 高效数据结构（SDS/ziplist/skiplist）。

**核心概念链**：纯内存操作（纳秒级）→ 单线程（无锁竞争+无上下文切换）→ epoll IO 多路复用（一个线程处理万级并发连接）→ SDS（简单动态字符串，O(1) 获取长度）→ ziplist/listpack（紧凑内存）→ skiplist（ZSet 底层，O(logN) + 简单实现）→ RDB（快照，COW fork）→ AOF（追加日志，三种 fsync 策略）→ 混合持久化（4.0+）。

**参考资料**：第十周/day4-5-Redis.md
