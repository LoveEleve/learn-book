# 第一个月 · 费曼检验

> **目标**：每月最后一个周末，随机抽取3个核心知识点，向一个完全不懂技术的人讲清楚。讲不出来 = 没学会。

## 检验规则
1. 找一个真人听众（推荐）或用录音设备
2. 对每个知识点进行15分钟讲解，不能用任何笔记
3. 听众必须能复述50%以上才算通过
4. 如果卡住了——回到材料重新学习，24小时后再来

## 本月3个核心知识点（随机抽取）

### 1. 虚拟内存
**讲述要求**：从一个Java程序 `new` 一个对象开始，讲清楚经历了什么——TLB→页表→物理内存→缺页中断→Swap。不能用术语，只能用"邮局地址"、"大仓库"之类的比喻。

**核心概念链**：虚拟地址 → TLB 查找 → 命中 → 物理地址；未命中 → 四级页表遍历 → 页不在内存 → 缺页中断 → 分配物理页 → Swap 换入 → 更新页表 → 返回物理地址。

**参考资料**：第一周/day5-虚拟内存.md、第三周/day3-4-虚拟内存.md

### 2. TCP 三次握手
**讲述要求**：不是说"SYN→SYN-ACK→ACK"，而是讲清楚为什么是三次不是两次也不是四次。用"打电话确认双方都能听见"的比喻，让听众听懂三次分别确认了什么。

**核心概念链**：第一次（客户端说"你在吗"）→ 服务端确认客户端的发送能力；第二次（服务端说"我在"）→ 客户端确认服务端的接收和发送能力；第三次（客户端说"好我知道了"）→ 服务端确认客户端的接收能力。两次无法确认双方收发，四次是多余的。

**参考资料**：第三周/day5-网络层.md、第三周/day6-7-TCP实战.md

### 3. Reactor 模式
**讲述要求**：从一个网络包到达网卡开始，讲清楚 epoll_wait→EventLoop→Handler 的完整链路。用"餐厅服务员"比喻：一个服务员（EventLoop）同时照看多桌客人（连接），哪桌需要服务就过去。

**核心概念链**：网卡收包 → DMA → Ring Buffer → 硬中断 → NAPI 软中断 → TCP 协议栈 → 放入 socket 接收队列 → sk_data_ready 唤醒 epoll → epoll_wait 返回 → EventLoop 分发 → Handler 处理请求 → 响应原路返回。

**参考资料**：第四周/day4-5-Reactor.md、第三周/day5-网络层.md
