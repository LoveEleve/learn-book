# 第一个月：根基（156h）

> **目标**：建立对计算机系统的完整底层认知，理解"程序如何运行"——从硬件到内核到网络。
> **核心理念**：地基不牢，后面的 JVM/MySQL/分布式都会浮于表面。
> **月末产出**：能手写 Reactor 服务器，能用 GDB/strace/tcpdump 分析程序行为，面试能讲清楚底层原理。
>
> **本规划依据**：[学习大纲.md](../学习大纲.md) 第一阶段（6-8周 → 精简为4周冲刺版）+ [4个月面试冲刺路线.md](../4个月面试冲刺路线.md)

## 学习方法提醒（来自学习大纲）

1. **先骨架后细节**：先通读建立脉络（第1遍），再精读核心章节并动手验证（第2遍）
2. **多书交叉印证**：同一主题可参考 2-3 本不同作者的书——例如进程管理可同时看陈海波（理论）、罗秋明（源码）、姜亚华（图解）三种视角
3. **动手验证优先**：每个概念都写代码/GDB 验证，操作系统和 JVM 部分配合源码阅读
4. **费曼学习法**：学完后用自己话讲一遍，讲不出来 = 没学会
5. **面试导向**：每个主题学完后问自己"面试官会怎么问这个问题"，准备 3 分钟回答
6. **不要贪多**：每阶段主攻 3-4 本核心书，其余书作为查阅/补充
7. **工具链建设**：熟悉 GDB/strace/perf/ftrace/eBPF 等工具，形成自己的问题排查工具箱
