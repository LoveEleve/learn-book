# 第五个月 · 费曼检验

> **目标**：每月最后一个周末，随机抽取3个核心知识点，向一个完全不懂技术的人讲清楚。讲不出来 = 没学会。

## 检验规则
1. 找一个真人听众（推荐）或用录音设备
2. 对每个知识点进行15分钟讲解，不能用任何笔记
3. 听众必须能复述50%以上才算通过
4. 如果卡住了——回到材料重新学习，24小时后再来

## 本月3个核心知识点（随机抽取）

### 1. 端到端性能优化全链路
**讲述要求**：从一个用户抱怨"这个页面好慢"开始，讲清楚你怎么一步步找到问题。用"医生看病"比喻：先问症状（用户感知）→ 量体温（监控指标）→ 拍 X 光（火焰图）→ 做血液检查（eBPF 追踪内核呼出）→ 开药方（代码优化/JVM 调优/系统参数调整）。

**核心概念链**：用户感知（响应时间 P99）→ 监控层（Prometheus + Grafana 指标看板）→ 应用层（JMH 基准测试、JIT 编译优化、代码热点分析）→ JVM 层（GC 日志、堆 dump、线程 dump）→ 系统层（perf 火焰图、strace 系统调用追踪）→ 内核层（eBPF / bpftrace 内核函数追踪）→ 硬件层（CPU 缓存命中率、内存带宽、磁盘 IOPS）。

**参考资料**：第十七周/day1-2-CPU性能分析.md、第十七周/day3-4-eBPF实战.md、第十八周/day1-2-Java性能深入.md、第十八周/day3-4-系统性能调优.md

### 2. eBPF 动态追踪原理
**讲述要求**：不从 kprobe/tracepoint 这些技术术语讲起，而是讲清楚"以前我们只能看日志，现在可以在程序运行时插入探测器，而且不会影响程序运行"。用"给高速行驶的汽车装传感器"比喻。

**核心概念链**：问题（生产环境不能停服务，日志不够细）→ eBPF 解决（内核虚拟机，安全执行沙盒化代码）→ kprobe（内核函数入口/出口探测）→ tracepoint（内核静态埋点）→ BPF Map（内核和用户态共享数据）→ 用户态读取（bpftrace 一行脚本 / BCC Python 工具）→ 安全保证（verifier 验证、JIT 编译、不能无限循环）。

**参考资料**：第十七周/day3-4-eBPF实战.md、第十七周/day5-可观测性.md

### 3. 算法解题通用思考框架
**讲述要求**：不是讲某一个算法题，而是讲"不管什么题，我的思考路径是一样的"。用"拆弹专家"比喻——先观察类型（定时/遥控/压力）→ 选工具包（DP/贪心/回溯/二分）→ 按步骤拆解（找最优子结构/找分治切割点）。

**核心概念链**：审题（输入/输出/约束条件）→ 分类（DP 信号：最大/最小/方案数；贪心信号：局部最优→全局最优；图信号：连通/路径/拓扑）→ 选数据结构（HashMap 查重、堆 TopK、单调栈 区间最值、并查集 连通性）→ 写代码前先说思路并确认 → 写完后主动测试（边界/空值/单元素）。

**参考资料**：第十九周/day1-2-算法冲刺一.md、第十九周/day3-4-算法冲刺二.md
