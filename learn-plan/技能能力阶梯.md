# 核心技能能力阶梯

> 每项技能的刻意练习路线图：基础→应用→熟练→精通。标注的阶段（M1W1 = 第一个月第一周）指该阶段完成后应达到的水平。

---

## GDB 调试

| 阶段 | 目标 | 里程碑 | 考核方式 |
|------|------|--------|---------|
| **基础**（M1W1） | 按 day 文件实验操作 | 会用 break/run/next/step/print/bt/ x/Ngx 查看栈帧和内存 | 完成 day1-2-体系结构.md 的 GDB 实验 |
| **应用**（M1W4） | 独立调试 Reactor 服务器，定位一次段错误 | 能从 core dump 找到崩溃的线程和调用栈 | 完成 错误诊断1 实验3 |
| **熟练**（M2W5） | 15 分钟内定位 JVM 堆中的一个对象 | 使用 `info vtbl` 查看 C++ vtable、`p` 查看对象布局 | 完成 day3-4-SATB标记.md GDB 验证 |
| **精通**（M5W17） | 教另一个人用 GDB 分析 core dump | 能反向（通过反汇编推测源码意图） | 费曼检验中讲解 core dump 分析 |

**核心命令速查**：
```
break <func>     # 断点
next/step        # 单步（不进入/进入函数）
continue         # 继续执行
print/x          # 打印变量/内存
bt full          # 完整调用栈
info threads     # 所有线程
info registers   # 寄存器
disassemble /m   # 反汇编+源码对照
x/4gx <addr>     # 以 8 字节 hex 显示 4 个单元
```

## perf / 火焰图

| 阶段 | 目标 | 里程碑 | 考核方式 |
|------|------|--------|---------|
| **基础**（M1W1 实验） | perf stat/record/report 基础命令 | 能用 `perf stat` 查看 IPC/分支预测率 | 完成 day6-7-动手实践.md perf 实验 |
| **应用**（M5W17） | 独立生成火焰图并分析 CPU 热点 | 从火焰图定位到具体函数和代码行 | 完成第十七周/day1-2-CPU性能分析.md 实验 |
| **熟练**（M5W18） | 给定一个未知 Java 应用，15 分钟内定位 CPU 瓶颈 | 结合火焰图+JIT 编译日志定位热点方法 | 完成 错误诊断3 实验2 |
| **精通**（M5W18+） | 对比不同版本/JVM 参数的火焰图做回归分析 | 建立性能基线，自动化火焰图对比 | 费曼检验中讲解端到端性能优化 |

**核心命令速查**：
```
perf stat -e cycles,instructions,cache-misses <cmd>
perf record -F 99 -p <pid> -g -- sleep 30
perf report --stdio
perf script | stackcollapse-perf.pl | flamegraph.pl > flame.svg
```

## strace

| 阶段 | 目标 | 里程碑 | 考核方式 |
|------|------|--------|---------|
| **基础**（M1W2） | 按 day 文件实验操作 | 会用 `strace -e trace=file` 追踪文件操作 | 完成第二周/day6-7-系统调用实战.md |
| **应用**（M1W4） | 独立追踪 Reactor 服务器的系统调用 | 对比 tcpdump 抓包和 strace 输出 | 完成 第二个月/第八周 网络追踪实验 |
| **熟练**（M2W8） | 追踪 Kafka 客户端的系统调用，分析零拷贝是否生效 | 能区分 sendfile / mmap+write / 普通 write | 完成第八周/day6-7-实战.md strace 实验 |
| **精通**（M5W17） | 用 `strace -c` 统计系统调用分布，定位性能瓶颈 | 自动分析 strace 输出发现异常系统调用模式 | 错误诊断3 性能问题排查 |

**核心命令速查**：
```
strace -e trace=open,read,write,close <cmd>
strace -c -p <pid>    # 统计模式
strace -f <cmd>        # 跟踪子进程
strace -T <cmd>        # 显示每次调用耗时
```

## bpftrace / eBPF

| 阶段 | 目标 | 里程碑 | 考核方式 |
|------|------|--------|---------|
| **基础**（M5W17） | 理解 eBPF 架构和 bpftrace 基础语法 | 能写出追踪 kprobe/tracepoint 的简单脚本 | 完成第十七周/day3-4-eBPF实战.md 实验 |
| **应用**（M5W17） | 用 bpftrace 追踪 JVM `futex` 锁竞争 | 写出生产级追踪脚本（含 histogram 统计） | 完成 第二个月/第八周 内核网络追踪实验 |
| **熟练**（M5W18） | 用 eBPF 追踪一次请求从网卡到应用的全链路延迟 | 结合 kprobe 和 tracepoint 拼出完整延迟分布 | 错误诊断3 全链路排查 |
| **精通**（M5W18+） | 编写 BCC Python 工具，发布到团队工具箱 | 实现自定义 eBPF 程序（非 bpftrace 脚本） | 费曼检验中讲解 eBPF 原理 |

**核心命令速查**：
```
bpftrace -l 'kprobe:tcp_*'
bpftrace -e 'kprobe:xxx { @[comm] = count(); }'
bpftrace -e 'tracepoint:xxx { printf(...); }'
```

## 学习路线总览

```
                   基础         →        应用        →        熟练        →        精通

GDB           M1W1 实验       M1W4 独立调试      M2W5 JVM验证       M5W17 教学
perf/火焰图    M1W1 实验       M5W17 独立生成     M5W18 快速定位     M5W18+ 自动化
strace        M1W2 实验       M1W4 网络追踪      M2W8 零拷贝验证    M5W17 性能诊断
bpftrace      M5W17 基础概念  M5W17 内核追踪     M5W18 全链路延迟    M5W18+ 自定义工具
JMH           M5W18 基础测试  M5W18 方案对照     M5W18+ JIT验证     M6 标准化框架
EXPLAIN       M3W9 读执行计划 M3W9 独立优化SQL   M3W10 复杂JOIN     M4 代价预估
Arthas        M2W7 基础命令   M5W18 生产诊断     M5W18+ ognl动态    M6 安全使用
Go pprof      M6W21 基础分析  M6W22 实战调试     M6W22 泄漏定位     M6+ 跨语言方法论
```

> **核心原则**：每个工具不是"学到会用"就够了，而是"能在面试中讲出我用这个工具解决过什么问题"——这才是真正的熟练。
