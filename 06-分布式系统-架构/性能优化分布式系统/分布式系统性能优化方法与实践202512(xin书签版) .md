<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">·资源竞争和死锁问题：不恰当的资源管理可能导致节点之间的资源竞争或死锁问题，</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">。数据存储和访问模式不合理问题：不合理的数据存储和访问模式可能导致数据穴余、</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">致数据的传输路径过长或通信拥塞，降低系统的效率。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">。数据一致性与性能之间的权衡问题：当多个节点同时访问或修改数据时，确保数据的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">一致性可能会导致性能问题。例如，不同节点之间的数据同步和复制可能会引发延</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">分布式系统，简而言之，是指以分散的方式利用分散的资源完成同一个或一组功能，例</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">如对等计算、文件共享等。随着网络需求的多元化和应用服务的复杂化，越来越多的业务开</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">始依赖分布式系统。这种趋势的背后是对高效性、可扩展性和容错性的迫切需求。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">在分布式系统中，各个节点相互协作，共同完成任务。同时，分布式系统能够根据实际</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">需求动态调整资源分配，实现负载均衡，确保业务的稳定运行。因此，分布式系统已经成为</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.80pt;color:#000000">现代信息技术领域的重要基石，广泛应用于云计算、大数据处理、物联网等多个领域。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:15.60pt;color:#000000">1.1.1分布式系统的定义</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">分布式系统是一个涵盖范围广泛的概念，可以从狭义和广义两个角度进行理解。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">从狭义上讲，分布式系统是指将系统的应用层、数据层或其他部分构建为分散的物理和</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">逻辑结构，这种结构通常呈现网状形态。例如，在分布式数据库系统中，数据的持久化层是</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">分散分布的，数据存储在不同的数据库实例中，这些数据库实例可以相互通信，并通过一套</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">综合管理系统维护数据的完整性和准确性。类似的分布式系统还有分布式文件系统、分布式</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.00pt;color:#000000">软件系统等。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">从广义上讲，分布式系统是指以分散的方式利用分散的资源来完成同一个或一组功能。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">这种意义上的分布式系统包括互联网、数据中心等，它们通过分散的资源协同工作，实现整</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.00pt;color:#000000">体的功能目标。</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">分布式系统的谬误最初由原Sun公司的彼得·多伊奇等人提出，指刚进人分布式计算领</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">域的工程师常常会做出的一系列错误假设。当工程师刚开始构建分布式系统时，难免会做出</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.90pt;color:#000000">8条假设、但这些假设经证明都是错误的，并且可能导致严重的后果。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.10pt;color:#000000">1）网络是可靠的。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">2）网络通信的传输延迟为零。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">3）网络通信的带宽是无限大的。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.40pt;color:#000000">4）网络通信是安全的。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">5）网络的拓扑结构不会改变。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.40pt;color:#000000">6）只有一位系统管理员。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">7）网络传输数据的成本为零。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.40pt;color:#000000">8）整个网络是同构的。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">这些错误假设至今仍会产生很大影响，架构师/程序员在设计分布式系统时应小心规</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">避，而且在当今的软硬件环境中它们又有了不同的变体。这些谬误让我们清楚地认识到，在</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">分布式系统中，错误是不可避免的，应该将相应的错误及异常处理方案融入分布式系统的架</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">构设计中，作为功能写入代码。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:16.50pt;color:#000000">1.1.4数据一致性与CAP理论</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">分布式系统是由多个分布在不同服务器节点上的子业务构成的，具有分布性、对等性、</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">并发性等特点。由于网络的不确定性，可能会出现通信异常等问题，进而导致分布式系统各</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">节点之间的数据一致性和可用性出现问题。CAP理论对数据一致性、可用性和系统分区之</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">间的关系进行了澄清和数学证明。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">1.数据一致性</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">在一个分布式系统中，由于网络延迟、节点故障等问题，数据同步可能会出现问题，这</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">时就需要使用一致性算法来保证数据的一致性。关于数据一致性对系统性能的影响和具体的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">实现方法，会在第
7章详细介绍。这里简要介绍Paxos和Raft算法，它们的目的都是在一</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">个分布式系统中保证数据的一致性。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">Paxos算法是由LeslieLamport在1998年提出的，是一种经典的分布式一致性算法。它</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">基于消息传递机制，通过在不同的阶段进行消息传递来达成一致性。Paxos算法的基本流程</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.70pt;color:#000000">是：提议者（Proposer）向多个接受者（Acceptor）发起提议，接受者对提议进行投票，并将</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.80pt;color:#000000">投票结果告知提议者，提议者根据接受者的投票结果最终确定一个值。</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:16.60pt;color:#000000">6第一部分基础理论</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:13.00pt;color:#000000">Raft算法是由DiegoOngaro和JohnOusterhout在2o13年提出的，是一种相对较新的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.60pt;color:#000000">分布式一致性算法。它是一种更易于理解和实现的算法，其基本思路是将分布式系统的状</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">态机复制到多个节点上，然后基于选举机制选出一个主节点来处理所有的客户端请求。在</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.80pt;color:#000000">Raft算法中，主节点负责接收客户端请求，并将请求复制到其他节点，从而保证数据的一</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:10.10pt;color:#000000">致性。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">Paxos算法和Raft算法虽然都是用于分布式系统的一致性算法，但它们之间存在一些区</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">别，具体包括以下几点：</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">。算法难度不同：Paxos算法相对复杂，理解起来较困难；Raft算法则相对简单，易于</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:10.80pt;color:#000000">理解。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">。日志复制方式不同：Paxos算法采用的是多数决策方式，即当多数节点接收到相同的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">决策时，就将这个决策写入自已的日志中；Raft算法采用的是领导人方式，即由领</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">导人负责将日志复制到所有节点中。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.80pt;color:#000000">·选举机制不同：Paxos算法中的领导者是由所有节点共同决定的，而Raft算法中的领</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.70pt;color:#000000">导者则是通过选举产生的。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">。数据同步方式不同：Paxos算法中的数据同步是异步的，即当一个节点提交一个请求</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">后，不需要等待其他节点的回应；Raft算法则是同步的，即当一个节点提交一个请</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">求后，需要等大多数节点回应后才能继续执行下一个操作。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.60pt;color:#000000">·可扩展性不同：Paxos算法在实际应用中难以扩展，而Raft算法则更容易扩展。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.80pt;color:#000000">综上所述，虽然Paxos算法和Raft算法都是用于解决分布式系统中的一致性问题，但</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">它们在算法难度、实现细节、可扩展性等方面存在一定的差异。在实际项目中，应根据特定</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">应用的需求和限制选择合适的算法。Paxos算法可以在一些极端情况下保证数据一致性，但</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">实现较为复杂，学习曲线较陡哨。对于一些对数据一致性要求较高、可以接受较为复杂系统</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">的场景，选择Paxos算法更加合适。Raft算法的实现相对简单，且易于理解和部署，适合那</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">些对数据一致性要求不是特别高，但对系统可用性和可靠性有一定要求的应用。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:15.80pt;color:#000000">O注</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">算法和协议是两个不同的重要概念。例如，Raft算法提供了实现分布式一致性的</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">方法和策略，而Raft协议则是实现Raft算法的具体规则和步骤。</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:18.70pt;color:#000000">1.2分布式系统的质量属性</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">分布式系统首先需要满足软件系统的功能需求，也就是满足用户的诉求，这是系统存在</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.70pt;color:#000000">的价值基础。然而，用户的满意度提升往往依赖于系统的质量。分布式系统主要有
10个质</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">量属性，体现了系统的不同质量特征。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">1）性能：系统与其环境交互以获得价值和避免损失的程度。性能与时间及软件系统满</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">足时间要求的能力有关，也与满足系统资源空间要求的能力有关。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.90pt;color:#000000">2）可伸缩性：随着用户或请求数量的增加，系统的运行和操作能力也随之增强。例</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:13.40pt;color:#000000">如，在云平台上，可伸缩性可以通过机器的水平或垂直扩展，也可以通过简单地附加</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">AutoScalingGroup属性来实现。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">3）可用性：它以正常运行时间的百分比来衡量，是指系统正常运行和正常工作的时间</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">比例。可用性受到系统错误、基础设施问题、恶意攻击和系统负载的影响。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">4）可扩展性：可扩展性度量系统的扩展能力和实现扩展所需的工作。扩展可以通过添</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">加新功能或修改现有功能来实现，但必须在不损害当前系统功能的情况下进行增强。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">5）一致性：一致性保证每个读操作均返回最近的写操作。这意味着在执行每一个操作</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">之后，所有节点的数据都是一致的，因此，无论它们连接到哪个节点，所有客户端都可以后</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">时看到相同的数据。一致性提高了数据的新鲜程度。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">6）弹性：系统可以从容地处理意外故障和恶意故障并恢复。检测故障并快速有效地恢</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.70pt;color:#000000">复是系统保持弹性是必要条件。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.60pt;color:#000000">7）易用性：系统的一种能力，能让用户安全、有效地执行任务，同时获得良好的用户</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">在理解分布式系统的性能指标之后，我们需要对目标分布式系统进行评估，通过获得的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">系统性能指标感知系统的性能现状，提出进一步提升系统性能乃至整体质量水平的方案并</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:10.20pt;color:#000000">实践。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">分布式系统的性能评估涉及多个方面，旨在确保系统在分布式环境中能够满足其设计目</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">标和用户需求。以下是分布式系统性能评估的一些关键指标：</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">。吞吐量（Throughput）：衡量系统在单位时间内能够处理的事务或请求的数量，用于</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.70pt;color:#000000">评估各个组件的处理能力以及系统的总体性能。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">。响应时间（ResponseTime）：衡量从发起请求到获得响应之间的时间，用于评估系统</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">的交互性能，无其是用户感知的延迟。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">。并发数量（Concurrency）：考察系统在同时处理多个请求或事务时的表现，确保系统</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.80pt;color:#000000">在高负载情况下依然能够高效地处理并发请求。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">·可伸缩性（Scalability）：评估系统在不同负载水平下的性能表现，确保系统能够有效</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">扩展以适应增加的工作负载，通过增加硬件资源或节点来提升性能。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">。可靠性和可用性（ReliabilityandAvailability）：衡量系统在面对故障或异常情况时的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:10.60pt;color:#000000">表现。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">。容错性：评估系统的容错性，确保系统在故障发生时能够及时恢复且保持高可用性。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">。网络性能（NetworkPerformance）：评估分布式系统中节点之间的通信性能，确保网</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">络延迟和带宽不会成为系统瓶颈，进而影响数据传输和通信效率。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">·资源利用率（ResourceUtilization）：监测系统中各个组件（如CPU、内存、磁盘等）</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.80pt;color:#000000">的资源使用情况，评估系统在不同工作负载下的资源利用效率，避免资源瓶颈和</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:10.60pt;color:#000000">浪费。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">。容量规划（CapacityPlanning）：预测系统在未来工作负载下所需的资源，通过容量规</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">划确保系统能够满足未来的业务需求而不出现性能问题。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">·安全性（Security）：评估系统在分布式环境中的安全性能，确保系统能够防范各种网</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.80pt;color:#000000">络攻击和安全威胁，同时不影响性能和可用性。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">这些指标共同构成了分布式系统的性能评估体系，帮助开发人员和系统管理员了解系统</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.70pt;color:#000000">在各种情况下的表现，并在必要时采取优化措施。</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:17.60pt;color:#000000">14.1可观测性</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">一般来说，可观测性是指根据观测到的外部输出对系统内部复杂状态或条件的理解程</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">度。一个系统的可观测性越高，我们就能越快、越准确地发现系统的性能问题并找到根本原</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">因，而不必进行额外的测试或编码。可观测性还指聚集、关联和分析分布式应用程序及其运</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">行所用硬件的稳定性能数据流的软件工具和实践，以更高效地监控、优化性能、排除故障和</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">调试该应用程序，达到满足客户体验预期、SLA和其他业务要求的目的。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">业界将可观测性能力划分为
5个层级，即指标、日志、追踪、告警和可视化分析，其中</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">指标与告警属于传统监控的概念范畴。触发告警的往往是明显的症状与表象，但随着架构与</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">应用部署方式发生变化，不告警并不意味着一切正常，因此，获取系统内部的信息就显得尤</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.50pt;color:#000000">为重要，而这些信息须借助“可观测性”能力中的主动发现能力。主动发现由排错、部析与</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.40pt;color:#000000">依赖分析三部分组成：</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.80pt;color:#000000">·排错（Debugging）：运用现有指标数据和信息去诊断故障出现的原因。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.20pt;color:#000000">·剖析（Profiling)：运用指标数据和信息来进行性能分析。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">。依赖分析：运用数据信息厘清系统之间的模块并进行关联分析。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">首先，无论是否发生性能问题，系统都会主动对运行情况进行诊断，通过指标呈现应用</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">运行的实时状态：其次，一且发现异常，逐层进行性能分析，调取详细信息，从而建立深入</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">的洞察；最后，利用模块与模块间的交互状态，通过链路追踪构建系统洞见。主动发现的</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.40pt;color:#000000">目的并不是为了告警与排障，而是通过获取全面的数据信息，构建对系统和应用架构最深</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.10pt;color:#000000">入的认知，从而帮助我们提前预测与防范故障或性能问题。一个可观测性平台的架构示例如</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:15.00pt;color:#000000">1.4.3面向性能指标的压力测试</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">性能测试主要通过模拟真实的业务负载来测试系统性能是否满足生产环境的要求。有</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">时，性能测试被泛化为压力测试、负载测试、强度测试、容量测试、大数据量测试、基准测</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.40pt;color:#000000">试、回放测试、稳定性测试等的总称。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">压力测试是一种测试方法，通过逐渐增加系统的工作负载，观察应用程序的性能变化，</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">以确定其在何种负载条件下会失效。此测试的目的是找出系统的瓶颈或不可接受的性能转折</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">点，以确定应用程序能提供的最大服务级别。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">压力测试的核心理念是不断增加压力，没有预设的性能指标。我们可以通过观察应用程</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.90pt;color:#000000">序何时崩溃，确定其瓶颈或不可接受的性能转折点，并确定最佳并发数。同时，也可以发现</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.50pt;color:#000000">由于资源不足或资源争用而产生的问题。</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.00pt;color:#000000">压力测试并不是在破坏系统，而是帮助我们更清晰地了解应用程序在崩溃时的表现。在</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">进行压力测试时，需要关注以下几点：</span>

<span style="font-family:'Droid Sans Fallback Regular';font-size:11.60pt;color:#000000">。应用程序是突然崩溃并退出，还是停止响应？</span>
<span style="font-family:'Droid Sans Fallback Regular';font-size:12.30pt;color:#000000">。停止压力测试后，应用程序能否恢复到之前的正常运行状态？如果不能，重启后能</span>
